Notes
-----
- 17s audio total wall ≈ 5s audio total wall: fixed cost (model load + Metal
  kernel JIT) dominates. The actual ASR encode/decode scales sub-linearly with
  audio length (encoder takes ~50ms per second of audio).
- Server path is 3-5x faster because model load + JIT happen once at server
  startup; per-request only encode + decode + HTTP overhead.
- All results well below REQ.md §5 target (30s audio ≤ 5s wall time).
- "Warm" sometimes equals or slightly exceeds "cold" because each invocation
  is still a fresh process — no Metal pipeline cache across runs. A persistent
  daemon would amortize this; v0.2 (real-time mode) will need that anyway.

  zig build
