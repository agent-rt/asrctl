# v0.8 报告 — Auto Backend Selection

> 目标：`asrctl --backend auto file.wav` 自动探测语言 → 中文走 qwen3，
> 其他语言走 whisper-large-v3-turbo。让用户不用关心后端就拿最优结果。
> 状态：✅ 通过（2026-05-02）。

## 演示

```sh
$ asrctl --backend auto -v samples/zh_short.wav
auto: detected language='zh' → backend=qwen3
backend: qwen3
今天天气真不错，我们去公园散步吧。

$ asrctl --backend auto -v samples/en_short.wav
auto: detected language='en' → backend=whisper
The quick brown fox jumps over the lazy dog. Hello world! This is a transcription test.
```

## 路由策略

```
zh / yue / wuu  →  qwen3        # 阿里 2026 SOTA on Chinese family
其他            →  whisper      # OpenAI multilingual SOTA
```

可调时机：用户给的样本若识别为 `ja` / `ko` / `cantonese` 等场景：
- 当前规则把 `yue`（粤语）和 `wuu`（吴语）也归 qwen3 — 它们属汉语方言族，qwen3 训练里覆盖更好。
- 日韩走 whisper — whisper 多语种训练量更大，目前看效果合理。

## 实现

asr.zig 加了一个独立 helper：

```zig
pub fn detectLanguage(allocator, model_path: tiny, samples: []f32) ![]u8 {
    var session = WhisperSession.open(.{ model_path, language=null });
    defer session.close();

    // 5s probe with audio_ctx=768 (v0.6 优化复用)
    var fparams = whisper_full_default_params(GREEDY);
    fparams.language = "auto";
    fparams.single_segment = true;
    fparams.audio_ctx = 768;
    whisper_full(ctx, fparams, samples[0..min(5*16000, n)]);

    return lang_str(whisper_full_lang_id(ctx));
}
```

main.zig transcribe 流程：

```
choice = parseBackend("auto")
if auto:
  decode wav once  (← 复用给后面真转录，省一次解码)
  tiny_path = ensureFile("ggml-tiny-q5_1.bin")
  lang = detectLanguage(tiny_path, samples)
  backend = backendForLanguage(lang)   // zh/yue/wuu → qwen3, else whisper
session = Session.open(backend, ...)
session.transcribePCM(samples)        // 复用同样的 PCM
```

## 性能（M2 Pro, ReleaseFast, warm cache）

| 样本 | explicit qwen3 | explicit whisper | --backend auto | auto 选 | 多出延迟 |
| --- | --- | --- | --- | --- | --- |
| EN (5.4 s) | 1668 ms | 2173 ms | 2185 ms | whisper | +12 ms |
| ZH (3.7 s) | 1163 ms | 2176 ms | 1684 ms | qwen3 | +521 ms |

理解：
- **EN → whisper auto +12 ms**：tiny 探针跑了 5s 音频但用了 v0.6 audio_ctx=768 + single_segment 优化，加上 wav 解码复用，几乎零开销。
- **ZH → qwen3 auto +521 ms**：要先 load tiny（~500 ms cold start），探针，free，再 load qwen3。这是切换后端付的"切换税"。
- 一次性付，单文件转录场景下完全可接受。多文件批处理时更明显——所以 batch 场景仍建议显式 backend。

## listen 不支持 auto 的原因

每个 utterance 都重探一次需要 ~50-100 ms 额外推理 + 切后端 + 重新加载 KV cache，对实时模式损害太大。明确报错：

```
$ asrctl listen --backend auto
error: --backend auto is not yet supported in listen mode
       (per-utterance language probe would add 50-100 ms latency).
       Workaround: pick qwen3 for Chinese-heavy use, whisper otherwise.
```

未来设计选项（不在 v0.8 范围）：
- 第一段探一次，后续 stick with：fragile（用户可能切语言）
- 持续运行 tiny 作为副探针：固定 ~5 % CPU 占用换 dynamic backend，工程量较大

## 模型下载

`--backend auto` 第一次跑会触发 `ggml-tiny-q5_1.bin`（31 MB）下载——和 v0.7 listen --partial 共用同一份缓存。已经用过 v0.7 的用户切 auto 不会重新下。

## 已知 limits

- 探针只跑前 5 s。如果音频前 5s 是静音/噪音/与剩余部分语言不同，会误判。
- 边界 case：code-switched 音频（中英混杂）会被识别为主导语言。当前实现没有重新评估机制。
- 没有人为 override：用户即便明确知道自己是中文也没有直接覆盖路由的方式（必须用 `--backend qwen3`）。但这正是 auto 的工作划分，OK。
