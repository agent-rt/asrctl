# v0.5 报告 — Whisper Streaming Partial Words

> 目标：在 listen 模式下，whisper backend 边说边出文字，不等 silence cut。
> 状态：✅ 通过（2026-05-02）。

## 体验

```sh
$ asrctl listen --backend whisper --partial
listening (Ctrl-C to stop, partial=on)…
                                              ← 用户开始说话
he                                            ← dim 灰字（partial）
hello wor                                     ← 同一行重绘
hello world                                   ← 同一行重绘
hello world how are you                       ← 仍在 partial
                                              ← 静音 600ms 触发 final
hello world, how are you?                     ← 同一行 commit + \n
                                              ← 下一句开始
nice to meet you                              ← partial
nice to meet you                              ← final commit
^C
stopped after 2 utterance(s)
```

效果：从用户开口到看到第一段文字 ~500ms-1s 延迟（取决于 `--partial-ms`），不必等到他说完。

## 实现

复用现有 `vad.Detector.segment` 累积缓冲——VAD 不只用来切段，也用来知道"现在 active 中的语音"。每 `partial_ms` (默认 500) 对 `detector.segment.items` 跑一次 `WhisperSession.transcribePCM`，重绘 stdout。

```zig
while (running) {
    usleep(50_000);
    capture.drain(&pcm_buf);
    detector.feed(allocator, pcm_buf, &ctx, onSegment);

    if (partial_enabled and detector.state == .active) {
        partial_iters += 1;
        if (partial_iters >= partial_target_iters and segment.len >= 16000) {
            partial_iters = 0;
            const result = session.transcribePCM(detector.segment.items);
            // \r move-to-col-0, \x1b[K erase-line, \x1b[2m dim, \x1b[0m reset
            print("\r\x1b[K\x1b[2m{s}\x1b[0m", .{result.text});
            ctx.partial_drawn = true;
        }
    } else {
        partial_iters = 0;
    }
}

// In onSegment final commit:
if (ctx.partial_drawn) write("\r\x1b[K");  // erase last partial
write(result.text); write("\n");
ctx.partial_drawn = false;
```

VAD 自带的 silence-cut 仍是最终提交 trigger；partial 只是中间预览。

## 输出协议

- **stdout 是 TTY**：partials 用 `\r\x1b[K\x1b[2m...\x1b[0m` 重绘 dim 文本，final 用 `\r\x1b[K{text}\n` 提交
- **stdout 非 TTY**（重定向 / 管道）：partial 自动禁用，`-v` 给 note；只有 final 输出
- **`-o file`**：file 只接 final，stdout 仍可 partial（如果 TTY）但当前 v0.5 也禁用，避免 partial 跟 -o 一起容易混淆。`asrctl listen -o log.txt --partial` partial 不显示。
- **`--backend qwen3 --partial`**：硬拒绝，返回 exit 1

## 性能开销

每次 partial 推理代价 ≈ encoder 重新跑整段 + decoder 跑相应文本。whisper-large-v3-turbo 在 M2 Pro 上 encoder ~30x realtime，所以 3s segment encode = ~100ms，5s = ~170ms。

按 `--partial-ms 500` cadence：每 500ms wall-time 跑一次 encode（100-200ms）= sustained 20-40% Metal GPU 占用 during speech。空闲期间归零。

trade-off：换来 latency 从"silence_ms (600) + 推理 (300-500)" → "partial_ms (500) + 推理在 background"，体感连续。

## CLI

```
asrctl listen --backend whisper --partial          # 默认 partial-ms=500
asrctl listen --backend whisper --partial --partial-ms 250   # 更密
asrctl listen --backend whisper --partial --vad silero      # 神经 VAD + 流式
```

## 限制 / 已知

- **qwen3 不支持 partial**：encoder-decoder 一次性架构，没 streaming 接口
- **`-o file` + `--partial`**：partial 静默禁用（避免文件里混 ANSI 噪音）
- **首字延迟**：必须积累 ≥1s 音频才开始 partial（whisper <1s 输入精度差）
- **partial 每次重新编码整段**：理论上 whisper.cpp 有 `whisper_full_with_state` 可复用 encoder 中间状态，但实现复杂度增加。v0.5 接受 redundant encode 成本
- **多说话人 / 长 monologue**：max_segment_ms (30s) 强制 cut，partial 体验保持
