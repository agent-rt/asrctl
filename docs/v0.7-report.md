# v0.7 报告 — Whisper 多模型 picker + 双模型 partial

> 目标：让 partial 用 tiny（63 ms）、final 用 large-v3-turbo（高质量），
> 同时给 power user 暴露 5 档 whisper 模型选择。
> 状态：✅ 通过（2026-05-02）。

## 卖点

`asrctl listen --backend whisper --partial` 现在体感"贴脸打字"——
partial latency 从 v0.6 的 ~530 ms 降到 ~63 ms。

实现机制：当 `--partial` 与 ≥ small 的 main 模型搭配时，自动并行加载一个
`tiny` (~31 MB) 模型专跑 partial 预览。final commit 仍用 main 模型保证质量。
两个模型同时驻留 ~580 MB RAM，trade-off 合理。

## 模型档位

| 名称 | 文件 | 大小 | 用途 |
| --- | --- | --- | --- |
| `tiny` | ggml-tiny-q5_1.bin | 31 MB | partial preview 默认 |
| `base` | ggml-base-q5_1.bin | 57 MB | |
| `small` | ggml-small-q5_1.bin | 181 MB | |
| `medium` | ggml-medium-q5_0.bin | 514 MB | |
| `large-v3-turbo` | ggml-large-v3-turbo-q5_0.bin | 547 MB | **default main** |

全部用 q5_0/q5_1 量化档位——质量损失对人耳/字幕场景几乎不可察，体积省一半。

## CLI

```sh
# 默认：main=large-v3-turbo + 智能选 tiny 做 partial
asrctl listen --backend whisper --partial

# 指定 main 模型
asrctl --backend whisper --whisper-model medium audio.wav

# 显式选 partial 模型（覆盖智能默认）
asrctl listen --backend whisper --partial --whisper-model large-v3-turbo \
              --whisper-partial-model base

# 关闭双模型，让 partial 用 main 模型（适合用户介意 RAM）
asrctl listen --backend whisper --partial --whisper-model medium \
              --whisper-partial-model medium
```

## 智能默认规则

```
if backend != whisper or not --partial:    无 partial 模型
if --whisper-partial-model 显式指定:       按用户值（与 main 同时不加载第 2 份）
elif main is small/medium/large-v3-turbo:  自动 partial = tiny
else (main is tiny/base):                  partial 复用 main
```

理由：
- main 是 small+ 时，单独加载 tiny ~31 MB 增量，换 5-16× partial 速度。
- main 是 tiny/base 时，本身就快，多加一份没意义。

## 性能实测

3 s wav 片段，warm cache，整进程 wall time（含 model load + Metal JIT，per-call 内部 encode/decode 是 partial 实际看到的开销）：

| model | wall | encoder | decoder | partial per-call (enc+dec) |
| --- | --- | --- | --- | --- |
| tiny | 660 ms | 44 ms | 19 ms | **63 ms** |
| base | 668 ms | 70 ms | 23 ms | 93 ms |
| small | 646 ms | 203 ms | 41 ms | 244 ms |
| medium | 1670 ms | 589 ms | 92 ms | 681 ms |
| large-v3-turbo | 1682 ms | 984 ms | 30 ms | 1014 ms |

partial latency 比较：
- v0.4（main 重跑）：~1014 ms
- v0.6（audio_ctx 缩放）：~530 ms (-48%)
- **v0.7（tiny partial）：~63 ms** (-94%)

体感：v0.4 → v0.7 ≈ "卡顿 1s 才出字" → "贴脸打字"。

## 实现要点

`WhisperSession` 加可选 `partial_ctx`：

```zig
pub const WhisperSession = struct {
    ctx: *c.whisper_context,           // main, used by transcribePCM
    partial_ctx: ?*c.whisper_context,  // optional smaller, used by transcribePCMQuick
    ...
};

pub fn transcribePCMQuick(self: *WhisperSession, samples: []const f32) Error!Result {
    const audio_ctx = clamp(...); // v0.6 audio_ctx scaling 仍然叠加生效
    const ctx = self.partial_ctx orelse self.ctx;  // ★ 偏好 partial
    return self.transcribeWithCtxAndModel(ctx, samples, audio_ctx);
}
```

main.zig 智能默认在 `listen` 入口决定 partial_size（参见 v0.7-report.md 上面的规则块），通过 `Options.whisper_partial_model_path` 传给 Session.open。

## 文件变更

- `src/main.zig`：`WhisperModelSize` enum + filename mapping + `isLargeForPartial`；`resolvePaths` 接 size；listen 智能默认
- `src/asr.zig`：`Options.whisper_partial_model_path`；`WhisperSession.partial_ctx`；`transcribeWithCtxAndModel` 内部接 ctx 参数
- `src/cli.zig`：`--whisper-model` 和 `--whisper-partial-model` flags

## 已知限制

- partial 用 tiny 时质量明显粗糙（"The Quick Brown Fox" 大小写错），但只是预览不影响 final。
- 双模型加载 ~580 MB RAM。低内存机器可以 `--whisper-partial-model <same>` 关掉。
- listen 模式已有 mic permission 弹窗 + isatty 校验，partial 模型只在真要 partial 时下载。
