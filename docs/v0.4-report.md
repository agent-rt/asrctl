# v0.4 报告 — 双 ASR 后端

> 目标：把 whisper.cpp 的转录 API 也接出来，asrctl 同时支持 Qwen3-ASR 和
> whisper-large-v3-turbo，同一个 CLI / VAD / listen / 文件路径。
> 状态：✅ 通过（2026-05-02）。

## 卖点

asrctl 现在是 **macOS 上当前 open-source 最强的 ASR CLI**：

- **Qwen3-ASR-0.6B**（阿里 2026 SOTA 多语 ASR，中文最强）
- **Whisper-large-v3-turbo Q5_0**（OpenAI multilingual SOTA）
- 同一个 CLI: `asrctl <wav>` / `asrctl listen` / `--server-url` 备用
- 同一个 silero VAD 神经分段（`--vad silero`）
- 6.2 MB 单二进制，完全静态（otool 只剩系统 framework）
- 无 Python 依赖

## CLI

```sh
asrctl --backend qwen3   audio.wav      # default; 1.5GB models, faster
asrctl --backend whisper audio.wav      # SOTA quality, 547MB model
asrctl --backend whisper --language zh audio.wav
asrctl listen --backend whisper --vad silero    # 全栈最强组合
```

## 实测对比（M2 Pro, ReleaseFast）

| 样本 | qwen3 | whisper | 差异 |
| --- | --- | --- | --- |
| EN short (5.4s) | 1.2 s | 2.2 s | qwen3 快 ~1s |
| EN long (17s) | 1.7 s | 2.1 s | 接近 |
| ZH short (3.7s) | 1.2 s | 2.2 s | qwen3 快 |
| ZH long (17s) | 1.7 s | 2.2 s | qwen3 快 |

**关键质量差异**（不是噱头）：

- ZH long「让我们坐在 _长_ 椅上」（whisper 正确） vs 「让我们坐在 _草_ 椅上」（qwen3 误识）
  → whisper 的 LM 上下文修正更强
- EN long「Qwen 3 ASR」：qwen3 漏掉 "ASR"（"Qwen-3 model"），whisper 字面拼写 "QEN 3ASR"
- ZH 标点：qwen3 用全角「，」，whisper 用半角「,」
- EN 标点：qwen3「Hello, world.」vs whisper「Hello world!」

互补性强：qwen3 快+中文强，whisper 上下文修正强+多语。

## 实现要点

### Session union dispatch

```zig
pub const Session = union(Backend) {
    qwen3: Qwen3Session,
    whisper: WhisperSession,
    // open / close / transcribePCM / transcribeFile 透传
};
```

`Qwen3Session` 是 v0.2 的旧 Session 重命名。`WhisperSession` 新写：
- `whisper_init_from_file_with_params(path, ctx_params)`，`use_gpu=true, flash_attn=true`
- `whisper_full(ctx, full_params, samples, n)` — 单段转录
- `whisper_full_n_segments` + `whisper_full_get_segment_text` 拼字符串
- `whisper_full_lang_id` + `whisper_lang_str` 拿 detected language

### 共享 ggml

两边都 pin 在 ggml 0.10.x：
- llama.cpp 的 ggml: 0.10.1 (commit 35cb684)
- whisper.cpp 的 ggml: 0.10.1 (commit 35cb684) — 同一个

`libwhisper.a` 不重定义 ggml 符号（v0.3 已验证 `nm -U` 无 `T ggml_*`），跟 llama.cpp 的 `libggml.a` 直接共存。链接顺序：
```
libmtmd.a → libllama.a → libwhisper.a → libggml.a → libggml-blas.a / -metal.a / -cpu.a / -base.a
```

### 模型管理

- qwen3: `ggml-org/Qwen3-ASR-0.6B-GGUF` （已有）
- whisper: `ggerganov/whisper.cpp/ggml-large-v3-turbo-q5_0.bin`（547 MB，量化精度损失极小）
- `asrctl model pull` 现在拉**两边**默认模型（用户用一遍后两个都在缓存）
- `--model PATH` 覆盖任何 backend 的主模型路径
- 缓存共享 huggingface_hub 标准 layout

### CLI 改动

- `--backend qwen3|whisper`（transcribe + listen 都加）
- `--language CODE`（whisper 用，qwen3 自动检测忽略此 flag）
- `--server-url` 仍只支持 qwen3（llama-server 不托管 whisper）；显式拒绝
- help 文案重写，把 backends 列在最前面

## 文件变更

- `src/asr.zig`：彻底重构。`Backend` enum + `Session` union + `Qwen3Session` + `WhisperSession`
- `src/wav.zig`：加 `decodeFileSimple`（io-less，给 WhisperSession 用）
- `src/cli.zig`：`--backend` `--language` flags
- `src/main.zig`：`resolvePaths` helper 处理两后端的模型查找；`transcribe` / `listen` 用同一个分发
- `bench/bench.sh`：新增 backend 维度（`--backend qwen3|whisper` 各跑 cold/warm）
- `bench/results-v0.4-dual.txt`：实测对比 + 质量差异分析
- `README.md`：双后端宣传位

## 二进制不爆

| | size |
| --- | --- |
| v0.3 (whisper.a 链入但未用) | 5.7 MB |
| v0.4 (whisper 全 API 用上) | **6.2 MB** |

Linker 在 ReleaseFast 仍然剥离了未引用代码：v0.4 真用了 `whisper_full` 等转录 API，所以多了 0.5MB。完全可接受。

## 已知 limits

- whisper 不支持 `--server-url`：没必要——whisper-large-v3-turbo 加载就 547MB，进程内已快。
- whisper streaming partial-word 没接：`asrctl listen` 跟 qwen3 一样靠 VAD 分段，每段独立 `whisper_full`。partial words 要 `whisper_full_with_state` + 进度回调，留 v0.5。
- `language CODE` 列表没枚举/校验：whisper 接受任何字符串，无效会自动 detect。
- whisper Q5_0 vs F16 质量差异未量化：F16 (1.5GB) 略好但慢，量化档位选择留给用户 `--model PATH`。

## 决定

v0.4 收。下一步候选：
- streaming partial words（whisper backend 的优势没充分发挥）
- 多 whisper 模型档位 picker (`--whisper-model tiny|base|medium|large-v3-turbo`)
- 自动 backend 选择（"看到中文样本默认 qwen3，英文默认 whisper"——但需要先识别才知道，鸡生蛋）
- WER bench 用真 LibriSpeech / Aishell 样本对比
