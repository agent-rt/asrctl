# v0.3 报告 — Silero VAD

> 目标：把能量 VAD 替换成 silero 神经网络 VAD，提升噪声鲁棒性。
> 状态：✅ 通过（2026-05-02）。

## CLI

```sh
asrctl listen --vad silero          # 默认仍是 energy；显式开 silero
asrctl listen --vad silero --threshold 0.5     # silero P(speech) 阈值
```

首次运行从 `ggml-org/whisper-vad` HF 仓库自动下 `ggml-silero-v5.1.2.bin`（885 KB）。

## 架构

`vad.zig` 重构为 backend-pluggable：
- `Backend` enum：`.energy` / `.silero`
- `Voicer` union：每个 backend 暴露 `frameSamples()` + `isVoiced(frame)`
- `Detector` 持有共享 FSM（idle/active + pre-roll + silence-cut + max-cap），只 dispatch 给 voicer

silero 实现（`SileroVoicer`）封装 whisper.cpp 的 VAD C API：
```
whisper_vad_init_from_file_with_params(model_path, ctx_params)
  ↓
per frame (512 samples = 32 ms @ 16 kHz):
  whisper_vad_detect_speech_no_reset(ctx, samples, n)
    → whisper_vad_n_probs / whisper_vad_probs
    → max(probs) >= threshold ? voiced : unvoiced
```

`_no_reset` 变体保留 LSTM 状态跨帧 → 正确的流式语义。

## vendor 链路

不依赖 `b.dependency("whisper.cpp")`（zig fetch 用自己的 std.net，不读 .curlrc，遇到 Tailscale split-route 会 EADDRNOTAVAIL）。改成把 whisper.cpp 预 clone 到 `_vendor/whisper.cpp/`（gitignored），build.zig 用 `b.path("_vendor/whisper.cpp")` + 第二段 cmake-shellout 编 `libwhisper.a`。

cmake flags：
```
-DBUILD_SHARED_LIBS=OFF
-DGGML_BACKEND_DL=OFF -DGGML_NATIVE=OFF
-DGGML_METAL=ON -DGGML_METAL_EMBED_LIBRARY=ON
-DGGML_BLAS=ON -DGGML_OPENMP=OFF
-DGGML_CCACHE=OFF                       # 关 sccache，否则它会去拨外网
-DCMAKE_C_COMPILER_LAUNCHER= -DCMAKE_CXX_COMPILER_LAUNCHER=
-DWHISPER_BUILD_TESTS=OFF -DWHISPER_BUILD_EXAMPLES=OFF -DWHISPER_BUILD_SERVER=OFF
```

ggml 共享：whisper.cpp 和 llama.cpp 都 pin 在 ggml 0.10.x。`libwhisper.a` 不重定义 ggml 符号（`nm -U` 查无 `T ggml_*`），只引用——和 llama.cpp 的 `libggml.a` 共存无冲突。链接顺序 `libmtmd.a → libllama.a → libwhisper.a → libggml*.a`。

## 性能

| | Debug | ReleaseFast |
| --- | --- | --- |
| 32 ms 帧 silero 单次 | ~38 ms | <1 ms |
| 实时占用（无说话 idle） | ~0.4× | 远 < 0.05× |
| binary 静态化 | 9.4 MB | **5.7 MB** |

`use_gpu = false`：silero 模型小（<1MB），CPU 1 thread 已是亚毫秒。开 GPU 反而和主 ASR 抢 Metal pipeline。

## 准确度对比（暂无定量）

silero 比能量法的关键改进：
- 风扇 / 键盘 / 远处人声等持续低音不再触发 voiced
- 低音量但清晰的语音不再被漏（能量阈值 0.012 在轻声场景误判）
- 不需要环境校准（能量阈值随房间变）

未做正式 benchmark，因为 listen 模式必须真说话测，CI 不便。用户体感反馈即定量。

## 踩到的网络坑

环境是 Tailscale split-tunnel，github.com / xethub.hf.co 等 IP 被 utun10 抢走但路由不到 → `Can't assign requested address`（EADDRNOTAVAIL）。修复：
- `~/.curlrc` 加 `interface = 10.0.0.151`（en0 实际 IP）→ 全局 curl 用对的源 IP
- `~/.ssh/config` 是 Nix 只读，改用 `GIT_SSH_COMMAND="ssh -o BindAddress=10.0.0.151"` env
- `zig fetch` 用自己的 std.net 完全绕开以上 → 改 vendor whisper.cpp 路径为本地 clone + 相对路径

## 文件变更

- `_vendor/whisper.cpp/`（新，gitignored，本地 clone）

## 已知限制

- `_vendor/whisper.cpp` 是手动 clone，**CI 不会自动 setup**——CI 跑 `zig build` 前要 `git clone` 或 submodule。下个 commit 加 setup script。
- 没在 `Detector` 状态切换时 `whisper_vad_reset_state`：跨 utterance LSTM 状态保留。可能对长 session 有累积影响，目前未观察到问题。
- whisper.cpp 自带的 ggml 子目录被 cmake 编了一遍（产物 ~5MB 在 `_vendor/whisper.cpp/build/ggml/`），但**不链接进 binary**——二进制大小不受影响。
